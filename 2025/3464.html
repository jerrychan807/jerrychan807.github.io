Git 工作流程指南

基本工作流程图

详细步骤说明

1. 克隆远程仓库

说明：将远程仓库完整复制到本地，包含所有分支和历史记录

2. 创建功能分支

说明：基于当前分支创建并切换到新的功能分支，隔离开发工作，不影响主分支

3. 修改代码并查看变更

说明：git diff 显示工作区与暂存区的差异，帮助你确认修改内容

4. 提交变更到本地仓库

说明：先用 git add 将修改添加到暂存区，再用 git commit 提交到本地仓库

5. 推送到远程仓库

说明：将本地功能分支推送到远程仓库，便于团队协作或代码审查

6. 处理主分支更新（当你开发期间主分支有更新时）

6.1 切换到主分支

说明：切换到主分支，准备获取最新更新

6.2 拉取主分支最新代码

说明：从远程仓库获取主分支最新代码并合并到本地主分支

6.3 切回功能分支

说明：切换回你的功能分支，准备合并主分支的更新

6.4 变基操作

说明：变基操作会先将你的修改"搁置"，然后应用主分支的最新提交，最后将你的修改重新应用在最新代码之上，使提交历史更加清晰

6.5 解决冲突（如果有）

如果出现冲突，Git会提示你解决冲突的文件。解决后：

说明：手动编辑冲突文件，保留需要的代码，然后继续变基过程

6.6 强制推送更新后的分支

说明：由于变基改变了提交历史，需要强制推送。注意：仅对个人功能分支使用强制推送

7. 完成功能开发后的清理

7.1 在GitHub上删除远程功能分支（通过Pull Request合并后）

7.2 切换到主分支

7.3 删除本地功能分支

说明：-D 参数强制删除分支，即使该分支还未合并

7.4 更新本地主分支

说明：确保本地主分支与远程保持同步，包含已合并的功能

Git 命令解释

命令 说明
git checkout -b xxx 创建并切换到新分支xxx。-b表示创建分支，checkout表示切换
git diff 显示工作区与暂存区的差异，帮助查看未暂存的修改内容
git add xxx 将xxx文件的修改添加到暂存区，准备提交
git commit 将暂存区的内容提交到当前分支的本地仓库
git push <远程主机名> <本地分支名> 将本地分支推送到远程主机的同名分支
git push -f <远程主机名> <本地分支名> 强制推送，会覆盖远程分支（谨慎使用）
git pull <远程主机名> <远程分支名> 从远程下载分支并与本地同名分支合并
git rebase xxx 变基操作，将当前分支的修改"重新应用"到xxx分支的最新状态之上
git branch -D xxx 强制删除本地分支xxx，即使该分支还未合并
